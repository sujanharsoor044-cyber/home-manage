Key Components in an SRS Document
System Objectives: To automate manual record-keeping of sales, purchases, and stock levels, improving efficiency and accuracy.
User Roles & Access:
Admin/Manager: Full control over employee data, inventory updates, supplier management, and financial reports.
Staff/Pharmacist: Capabilities for searching products, creating invoices, and processing sales.
Functional Requirements:
Inventory Management: Tracking medicine stock, expiry dates, batch numbers, and supplier details.
Billing System: Generating invoices for customers, calculating totals, and managing payment records.
Database Management: Secure storage for medicine, customer, and employee records.
Reporting: Generation of daily, weekly, or monthly sales and inventory reports.
Non-Functional Requirements:
Security: Password-protected access and secure data handling.
Reliability & Usability: A user-friendly interface that ensures data accuracy and easy access.
System Scope: Often covers both desktop-based systems for in-store management and online modules for customer ordering. 
Potential Technologies Identified
Frontend/Backend: Java, PHP
Database: MySQL, SQL Server
